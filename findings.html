The Three Questions

From these finding, I saw that the website is mostly good, with only one real issue. The real issue is that it does not display error in the code in the demos or in the playground page.

This means that almost everything else about the site is clear and easy to use and easy to learn from. There are some things that might be changed (like how Ali said it would be a good idea to make that part of the navagation bigger so people can notice it more), but nothing urgent or that would make the website unusable to the user.

What needs to be done is that the error messages should be implamented and the developer can think about implementing the slight issues with the navagation.
